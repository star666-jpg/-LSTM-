## 一、数据模块 `src/data/`

### 1.1 技术指标计算 `indicators.py`

从茅台历史行情（OHLCV）计算 15 维技术指标，覆盖趋势、动量、波动、成交量四个维度。

| 指标 | 类别 | 含义 |
|------|------|------|
| SMA / EMA | 趋势 | 简单/指数移动平均，反映基础趋势方向 |
| MACD / 信号线 | 趋势 | 长短期EMA差值，判断中长期趋势变化 |
| CCI | 趋势 | 价格偏离统计均值的程度，>100为超买 |
| ADX | 趋势 | 趋势强度（不区分多空） |
| RSI | 动量 | 近14天涨跌力量对比，衡量超买超卖 |
| Stochastic | 动量 | 收盘价在近期波动区间中的相对位置 |
| ROC | 动量 | 当前与N天前价格的变化率 |
| 布林带上下轨 | 波动 | 均线±2倍标准差，判断极端行情 |
| ATR | 波动 | 平均真实波幅，综合考虑跳空缺口 |
| MFI | 成交量 | 含成交量的RSI，判断资金流入流出 |
| OBV | 成交量 | 能量潮，用量能确认价格趋势 |

**核心函数：**
```python
from src.data.indicators import build_indicator_dataset

df = build_indicator_dataset(
    price_csv="data/raw/moutai_price.csv",
    output_csv="data/processed/moutai_technical_indicators.csv",
)
```

### 1.2 数据预处理 `preprocess.py`

滑窗切分 + MinMax 标准化 + 数据集划分。

**划分比例**（参照孙惠莹2025 §3.2）：训练 60% / 验证 15% / 测试 25%

```python
from src.data.preprocess import split_dataset

splits = split_dataset(
    df,
    feature_cols=feature_cols,
    window=64,    # 每个样本用过去64天的数据预测第65天
    horizon=1,
)
# splits["X_train"]  形状: (N, 64, n_features)
# splits["y_train"]  形状: (N,)  归一化的次日收盘价
# splits["scaler"]   MinMaxScaler，用于反归一化结果
```

**注意：** scaler 只在训练集上 fit，验证/测试集用 transform，防止数据泄露。

## 二、情感模块 `src/sentiment/`

### 2.1 情感打分 `snowlp_scorer.py`

使用 SnowNLP 对股吧评论进行情感分析，聚合为日级别 7 维特征。

**处理流程：**
```
原始评论文本
    ↓ clean_text()：去除URL、表情符[哭泣]、噪声字符
    ↓ score_text()：SnowNLP 输出 [0,1] 情感分
    ↓ aggregate_daily_sentiment()：按日聚合为7维特征
    ↓ merge_with_indicators()：与技术指标按交易日对齐
```

**7维情感特征：**

| 特征名 | 含义 | 取值范围 |
|--------|------|---------|
| `sentiment_mean` | 日均情感得分 | [0, 1]，>0.5 偏正向 |
| `sentiment_std` | 情感分歧度 | ≥0，越高说明多空分歧越大 |
| `positive_ratio` | 正向评论占比（得分>0.6） | [0, 1] |
| `negative_ratio` | 负向评论占比（得分<0.4） | [0, 1] |
| `comment_count` | log(评论数)，反映市场关注度 | ≥0 |
| `sentiment_momentum` | 与前一交易日均值之差 | 正=情绪改善，负=情绪恶化 |
| `sentiment_ma5` | 5日情感均线，过滤短期噪声 | [0, 1] |

**节假日处理：** 若某交易日无评论数据，用前一交易日数据前向填充；开头冷启动期填中性值（0.5）。

## 三、模型模块 `src/models/`

### 3.1 支路一：TCN-LSTM-Transformer `tcn_lstm_transformer.py`

**参照：** 孙惠莹(2025) §3.1 · LSTM[3] · TCN[5] · Transformer[7] · 注意力机制[8]

三阶段时序特征提取，各阶段互补：

```
输入 (B, T=64, F=20+7)
    ↓
TCN（4层膨胀因果卷积）
  - 膨胀因子: 1, 2, 4, 8
  - 感受野: 2×(2⁴-1)×1+1 = 31 步
  - 因果性：只能看到当前及之前的时间步（无未来泄漏）
    ↓
4层堆叠 LSTM（隐藏维度 256）
  - 捕捉短期有序依赖和门控记忆
    ↓
位置编码 + Transformer Encoder（4头，2层，Pre-LN）
  - 捕捉序列中任意位置之间的全局关系
    ↓
LayerNorm + Linear → 输出 (B, 128)
```

**关键设计：**
- 因果卷积：手动左侧填充，避免"看到未来"
- Pre-LN（norm_first=True）：比 Post-LN 训练更稳定
- 位置编码：正弦/余弦编码，让 Transformer 感知时序位置

---

### 3.2 融合层：H-MoE `hmoe.py`

**参照：** 孙惠莹(2025) §3.3 · MoE原始[22] · 稀疏MoE[20] · 负载均衡损失[21]

两级层次化路由，共 4 个叶节点专家：

```
输入特征 (B, 128)
    ↓
第一级门控（粗粒度）
  → 多头方向权重 w₀
  → 空头方向权重 w₁
    ↓
第二级门控（细粒度，各自独立）
  多头分支 → [强趋势上涨, 震荡上涨]
  空头分支 → [强趋势下跌, 震荡下跌]
    ↓
叶专家权重 = w_super × w_sub（Top-K=2 稀疏化）
    ↓
加权汇总专家输出
    ↓
  ┌──────┴──────┐
回归头（次日收盘价）  分类头（涨跌概率）
```

**关键机制：**

| 机制 | 作用 |
|------|------|
| Top-K 稀疏路由（K=2） | 每次只激活最相关的2个专家，减少计算量 |
| 训练时噪声注入 | 鼓励门控探索，避免过早收敛到固定专家 |
| 负载均衡辅助损失 | 防止所有样本塌缩到同一专家 |
| GELU + LayerNorm | 比 ReLU 更平滑，训练更稳定 |

**训练时总损失：**
```python
loss = mse_loss(pred_price, true_price) + out["aux_loss"]
```
`aux_loss` 系数默认 0.01，可在 `HMoE(aux_loss_coef=...)` 中调整。

## 四、训练脚本 `scripts/`

### 4.1 阶段一训练 `train_stage1.py`

支路1（TCN-LSTM-Transformer）+ H-MoE，仅用时序数据验证基线。

```bash
python scripts/train_stage1.py

# 自定义参数
python scripts/train_stage1.py --epochs 100 --window 30 --lr 0.0005 --batch 16
```

**训练流程：**
1. 加载茅台行情 → 计算技术指标 → 滑窗切分
2. 每轮：前向传播 → MSE损失 + aux_loss → 反向传播 → 梯度裁剪
3. ReduceLROnPlateau 学习率调度（验证集不改善则 ×0.5）
4. Early Stopping（默认 patience=10）
5. 测试集：反归一化后计算 MSE / MAE / R²，与论文基准对比

**论文基准对比：**

| 模型 | R² |
|------|----|
| 单体 LSTM（孙论文基线） | 0.42 |
| TCN-LSTM-Transformer（孙论文第三章） | 0.68 |
| 完整模型含情感+H-MoE（孙论文最终） | 0.71 |

## 五、配置文件 `configs/config.yaml`

所有超参数集中管理，训练脚本会自动读取：

```yaml
data:
  window: 64        # 滑窗长度（孙惠莹第四章实验设置）
  split_ratio: [0.60, 0.15, 0.25]

model:
  feat_dim: 128     # 各支路统一输出维度
  lstm_hidden: 256  # LSTM 隐藏维度

training:
  epochs: 50
  batch_size: 32
  lr: 0.001
  early_stopping_patience: 10
```
