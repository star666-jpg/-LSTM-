### 四、训练实跑结果与诊断

| 设置 | 数据范围 | 测试 R² | 测试 MAE |
|------|----------|--------|---------|
| phase1（5维 OHLCV，70/30 划分） | 2018-2023 | -0.42 | 156 元 |
| stage1（19维指标，60/15/25 划分） | 2018-2023 | -14.23 | 525 元 |
| stage3（19维 + BERT，60/15/25） | 2018-2023 | -29.93 | 829 元 |

**为什么 R² 都是负的？** 数据分布漂移：

- 训练集（前 60-70%）覆盖 2018-2021，含 2021 年茅台峰值（~2700 元）
- 验证/测试集（后 25-30%）落在 2021 之后的下行段（~1500-1900 元）
- `MinMaxScaler` 仅 fit 在训练集上，test 数据本身在分布内，但**价格走势模式**
  （持续上行 → 高位震荡下行）训练时从未见过，模型学到的"线性外推"在 test 段彻底反向

**与 memory 中 `phase1 R²=0.70` 的差异：** memory 记录的结果是早期数据切片上跑的，
当时 test 段恰好仍在上行通道里。本次重新拉取覆盖到 2026 年的全量数据后，
分布漂移问题完全暴露。

**修复方向（待做，未在本次提交内）：**
- 改用收益率（log return）替代原始价格做目标，去除非平稳性
- 或把训练窗口滚动地推到测试窗口前一天（walk-forward），而非一次性切分
- BERT 文本支路目前只用了 10 条 akshare 新闻 + "无相关新闻"占位，
  对训练几乎没有信号贡献，需修复股吧爬虫后再评估文本支路真实增益

### 支路二：TIFS 图像 + CS-ACNN

**`scripts/generate_tifs.py`**（预计算，只跑一次）

将技术指标时序数据编码为 RGB 图像，三通道对应三种语义：
- Channel R（价格/趋势，11维）：OHLCV + SMA/EMA/MACD/CCI/ADX

每个滑窗 (T, F) → 逐列 MinMax → `torch.nn.functional.interpolate` bilinear resize → (3, 64, 64)
输出 `data/processed/tifs_images.npy`（n_windows, 3, 64, 64）

**`src/data/tifs_dataset.py`**

`TIFSDataset` + `load_tifs_splits()` 一站式加载 train/val/test。

**`src/models/cs_acnn.py`**（已有骨架，已验证）

CS-ACNN 含通道注意力（SE-Net风格）+ 空间注意力（CBAM风格），
输出 multi_scale 列表（供 INSA 早期融合）和 deep_feat (B, 128)。

### 三、阶段三训练脚本 `scripts/train_stage3.py`

**模型架构（Stage3Model）：**
```
ts_window  (B, T, ts_dim) → TCNLSTMTransformer → ts_feat  (B, 128)
bert_window(B, T, 768)    → GFM 逐步融合      → text_feat (B, 128)
                              concat (B, 256)
                                  ↓
                          HMoE(in_dim=256)
                           ↓          ↓
                         price     direction
```

关键设计：
- BERT 权重不参与训练（预计算 CLS），GFM 权重参与训练，大幅降低显存需求
- BERT CLS 向量在输入 GFM 前做 L2 归一化，稳定梯度
- `TextStockDataset`：每个样本是 (ts_window, bert_cls_sequence, label_price, label_dir)
- 损失 = 0.5 × MSE回归 + 0.5 × BCE分类 + H-MoE负载均衡辅助损失
- ReduceLROnPlateau 以 -R² 为监控指标，Early stopping patience=10


### 二、支路一：TCN-LSTM-Transformer `src/models/tcn_lstm_transformer.py`

实现时序建模主干网络，三阶段串联：

**TCN（时序卷积网络）**
- 4层膨胀因果卷积，膨胀因子 1→2→4→8
- 感受野约 31 步，能捕捉长程时序依赖
- 关键修复：改用手动左侧填充，确保因果性（输出位置 t 只能看到 t 及之前的输入）

**LSTM**
- 4层堆叠，隐藏维度 256
- 捕捉短期有序依赖

**Transformer Encoder**
- 新增正弦/余弦位置编码（时序任务必须项）
- 4头自注意力，Pre-LN 结构（训练更稳定）
- 捕捉序列中任意位置之间的全局关系

输出：最后时间步特征向量 `(B, 128)`，供 H-MoE 使用。

---

### 三、融合层：H-MoE `src/models/hmoe.py`

实现层次化混合专家模块，对应孙惠莹(2025) §3.3：

**两级路由结构（共4个叶专家）：**
```
第一级（粗粒度）：多头方向 / 空头方向
第二级（细粒度）：强趋势上涨 / 震荡上涨 / 强趋势下跌 / 震荡下跌
```

**关键机制：**
- Top-K 稀疏路由（K=2）：每次只激活最相关的2个专家
- 训练时噪声注入：鼓励门控探索不同专家
- 负载均衡辅助损失（引自 Switch Transformer）：防止专家坍缩
- 双输出头：回归头（收盘价）+ 分类头（涨跌方向）

### 四、情感打分模块 `src/sentiment/snowlp_scorer.py`

基于 SnowNLP 对股吧评论进行中文情感分析：

- 文本清洗：去除 URL、表情符 `[哭泣]`、噪声字符
- 批量打分：逐条评论输出 [0,1] 情感分
- 7维日级别特征聚合：均值、标准差、正负比例、评论量、情感动量、5日均线
- 交易日对齐：节假日数据前向填充，冷启动期填中性值 0.5
