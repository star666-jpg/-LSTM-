## 一、我们在做什么

**一句话**：以贵州茅台（600519）为研究对象，把行情时序、K 线图像、中文新闻、股吧情感四种模态的数据
"喂"进一个统一的神经网络，让它同时给出**次日收盘价**（回归）和**涨跌方向**（分类）两个预测。

这不是单纯做一个 LSTM——而是要复现并融合两篇论文的方法，再尝试超越它们的基准：

| 论文 | 主要贡献 | 我们用了什么 |
|------|----------|--------------|
| 孙惠莹（2025） | TCN-LSTM-Transformer 时序基线 + H-MoE 融合 | 支路 1 + 融合层 |
| 昝泓含（2024） | TIFS 技术指标图像编码 + CS-ACNN | 支路 2 |

在论文基础上**额外引入**：
- 中文新闻 BERT 特征 + GFM 门控（支路 3）
- 股吧标题 SnowNLP 情感打分（支路 4）

## 三、模型怎么搭

```
输入                    支路（特征提取）                         融合           输出
─────────────────────────────────────────────────────────────────────────────────────
OHLCV+技术指标 (T,19) → 支路1: TCN+LSTM+Transformer → ts_feat (128)
                                                                  ╲
新闻标题 (T,~)        → 支路3: BERT-Base-Chinese + GFM → text_feat (128) ─┐
                                                                          ├→ HMoE → price (回归)
TIFS 蜡烛图 (3,64,64) → 支路2: CS-ACNN（CBAM 注意力）  → img_feat  (128) ─┤        ↘ direction (分类)
                                                                          │
股吧情感分              → 支路4: 7 维特征拼入支路1 输入                   │
                                                                          │
                                                    concat (B, 256/512) ──┘
                                                                          │
                                                    H-MoE: 第一级粗粒度 (多/空头)
                                                          ↓
                                                    第二级细粒度 (4 个叶专家)
                                                          ↓
                                                    Top-K=2 稀疏路由
                                                          ↓
                                                    回归头 + 分类头
```

### 关键设计

- **TCN-LSTM-Transformer 串联**（而非并联）：TCN 抓局部高频模式，LSTM 编码长程依赖，Transformer 用注意力做全局重权
- **BERT 冻结，只训 GFM**：节省显存，BERT CLS 向量预计算后缓存到 `bert_cls_daily.npy`
- **H-MoE 层次化路由**：模拟"先判断市场风格（牛/熊），再选具体策略（4 个专家）"，用 Top-K=2 稀疏化让单样本只激活 2 个专家
- **双任务联合训练**：`loss = 0.5·MSE + 0.5·BCE + 0.01·aux`（aux 是 H-MoE 的负载均衡损失）

## 六、为什么 R² 是负的——数据漂移诊断

这是本次最重要的发现。

### 现象

在本次新拉的 2018-2023 完整数据上跑：

| 设置 | 划分 | 测试 R² | 测试 MAE |
|------|------|--------|---------|
| phase1 (5 维 OHLCV) | 70/30 | -0.42 | 156 元 |
| stage1 (19 维指标) | 60/15/25 | -14.23 | 525 元 |
| stage3 (19 维 + BERT) | 60/15/25 | -29.93 | 829 元 |

**全部为负**，意味着这些模型不如简单输出训练集均值。

### 根因

茅台的价格走势：
- 2018-2021：**单调上行**，从 ~440 元涨到峰值 ~2700 元
- 2021 之后：**高位下行**，跌到 ~1500 元

按时间顺序切分：
- 训练集（前 60-70%）= 2018 到 2021 上行段，**只见过涨**
- 测试集（后 25-30%）= 2021 之后下行段，**只见过跌**

`MinMaxScaler` 用训练集 fit 得到 `min=440, max=2700`，归一化后 close 在 [0,1]；
**测试集的 close 值在分布内**（0.4-0.7），但**走势模式从未在训练集出现**。
模型隐式学到"价格倾向于继续上涨"，在测试段彻底反向 —— 这就是非平稳金融时间序列的经典陷阱。

### 与 memory 里"phase1 R²=0.70"的差异

之前的 R²=0.70 是**早期数据切片**上跑出来的，当时测试段恰好仍处于上行通道。
本次重新拉到 2026 年的全量数据，分布漂移完全暴露。

### 修复方向（未在本次提交内）

| 方向 | 思路 | 工作量 |
|------|------|--------|
| **改预测目标** | 预测 log return（Δlog(price)）而非原始价格，去除非平稳性 | 中 |
| **walk-forward 切分** | 滚动地用过去 N 天预测下一天，每次重新 fit scaler | 中 |
| **缩短数据范围** | 只用走势风格相近的窗口（比如只 2018-2020） | 小 |
| **修股吧爬虫** | 对接东方财富新接口（需逆向） | 中-高 |
